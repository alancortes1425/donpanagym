🌐 1. ¿Qué significa publicar una página web?

Publicar una página web significa subir todos los archivos de tu sitio (HTML, CSS, imágenes, etc.) a un servidor conectado a Internet, para que cualquier persona con acceso a la red pueda visitarlo desde cualquier parte del mundo.

🎯 Objetivo: Hacer que tu sitio web sea accesible públicamente a través de una URL (dirección web) como https://tusitio.github.io o un dominio personalizado.
🌍 Página local: Solo visible en tu computadora (archivos en tu disco duro).
🌐 Página publicada: Alojada en un servidor, accesible desde cualquier dispositivo con Internet.
🔗 URL pública: Dirección única que usan los visitantes para acceder a tu sitio.

💾 2. Hosting y Dominio: conceptos fundamentales

🏠 Hosting (Alojamiento) Es el servicio que proporciona espacio en un servidor para almacenar los archivos de tu sitio web. Es como alquilar un terreno donde construir tu casa digital.
🔖 Dominio Es el nombre único que identifica tu sitio web (ej: artesaniasdonpepe.com). Es como la dirección de tu casa.
Tipo de HostingDescripciónPrecio
Hosting Gratuito Espacio limitado, subdominio incluido. Ideal para proyectos escolares. Gratis
Hosting Compartido Varios sitios comparten un mismo servidor. Bueno para sitios pequeños. $3-10 USD/mes
VPS / Dedicado Servidor exclusivo. Mayor rendimiento y control. $20+ USD/mes
✨ GitHub Pages: Es un servicio gratuito de hosting que ofrece GitHub para publicar sitios web estáticos directamente desde repositorios. Perfecto para proyectos educativos.

🐙 3. GitHub Pages: hosting gratuito y profesional

GitHub Pages es un servicio de hosting gratuito que permite publicar sitios web estáticos directamente desde repositorios de GitHub. Es ideal para proyectos personales, portafolios y páginas de documentación.

🎁 Ventajas de GitHub Pages:
  • ✅ Completamente gratuito
  • ✅ Sin publicidad
  • ✅ Soporta HTML, CSS, JavaScript
  • ✅ URL personalizable: usuario.github.io/nombre-repo
  • ✅ Integración con Git (control de versiones)
  • ✅ SSL/HTTPS automático (seguridad)
GitHub Pages

📌 URL de publicación:

https://tu-usuario.github.io/nombre-del-repositorio/

Ejemplo: https://juanperez.github.io/artesanias-don-pepe/

📁 4. Preparar los archivos para publicar

Antes de publicar, debemos asegurarnos de que nuestro sitio web esté correctamente organizado y funcione correctamente de forma local.

📌 Reglas importantes:
  • El archivo principal debe llamarse index.html (es la página de inicio)
  • Todas las rutas a imágenes, CSS y otros archivos deben ser relativas (no usar rutas absolutas como C:\misitio\imagen.jpg)
  • Los nombres de archivos y carpetas deben usar minúsculas, sin espacios ni caracteres especiales (ñ, acentos, etc.)
  • Estructura organizada: carpetas para css/, img/, js/ (si aplica)
mi-sitio-web/ │ ├── index.html # Página principal (OBLIGATORIO) ├── productos.html # Otras páginas ├── contacto.html │ ├── css/ │ └── estilos.css # Hoja de estilos principal │ ├── img/ │ ├── logo.jpg │ ├── producto1.jpg │ └── banner.jpg │ ├── js/ # (opcional) │ └── script.js │ └── README.md # Documentación (opcional)

📋 5. Pasos para publicar con GitHub Pages

1

Crear una cuenta en GitHub

Ingresa a https://github.com y regístrate. Usa un correo electrónico válido y elige un nombre de usuario (será parte de tu URL).

2

Crear un nuevo repositorio

Haz clic en el botón verde "New repository" o "+" → "New repository".

  • Repository name: Elige un nombre (ej: artesanias-don-pepe)
  • Description: (opcional) Breve descripción del proyecto
  • Public / Private: Selecciona Public (para GitHub Pages gratuito)
  • Initialize with README: Puedes marcarlo para crear un README inicial
  • Haz clic en "Create repository"
3

Subir los archivos del sitio web

Hay dos formas de subir archivos:

Opción A (recomendada): Usar la interfaz web de GitHub

  • Haz clic en "Add file" → "Upload files"
  • Arrastra todos los archivos y carpetas de tu proyecto
  • Escribe un mensaje de commit (ej: "Subir sitio web")
  • Haz clic en "Commit changes"

Opción B: Usar Git desde la terminal (para usuarios avanzados)

git init git add . git commit -m "Primera versión del sitio" git remote add origin https://github.com/tu-usuario/nombre-repo.git git push -u origin main
4

Activar GitHub Pages

  • En tu repositorio, ve a la pestaña "Settings" (Configuración)
  • En el menú lateral izquierdo, busca "Pages"
  • En la sección "Branch", selecciona main (o master) y la carpeta /(root)
  • Haz clic en "Save"
  • Espera unos minutos mientras GitHub construye y publica tu sitio
5

Obtener la URL pública

Una vez activado, verás un mensaje con la URL de tu sitio publicado:

https://tu-usuario.github.io/nombre-del-repositorio/

¡Comparte este enlace con tu docente y compañeros!

⏱️ Tiempo de publicación: Puede tomar entre 1 y 5 minutos para que los cambios se reflejen. Si no ves tu sitio, espera un poco y actualiza la página.

✅ 6. Verificar que el sitio funciona correctamente

Después de publicar, es importante verificar que todo funcione correctamente:

🔗 1. Verificar enlaces Asegúrate de que todos los enlaces del menú y los hipervínculos funcionen correctamente.
🖼️ 2. Verificar imágenes Comprueba que todas las imágenes se carguen correctamente.
📱 3. Verificar responsividad Prueba tu sitio en diferentes dispositivos o usando herramientas de desarrollador.
🔒 4. Verificar HTTPS GitHub Pages proporciona SSL automático. Asegúrate de que la URL comience con https://.
=== LISTA DE VERIFICACIÓN POST-PUBLICACIÓN === □ 1. El archivo index.html está en la raíz del repositorio □ 2. La URL se abre correctamente en el navegador □ 3. Todos los enlaces (menú, botones) funcionan □ 4. Todas las imágenes se visualizan correctamente □ 5. Los estilos CSS se aplican correctamente □ 6. El sitio se ve bien en móvil (probar con DevTools) □ 7. No hay errores 404 (página no encontrada) □ 8. La URL es accesible desde otro dispositivo (opcional)

ProblemaPosible solución
El sitio no se ve después de publicarEspera 5 minutos y actualiza. Verifica que GitHub Pages esté activado en Settings → Pages.
Las imágenes no se carganVerifica que las rutas sean correctas. Usa rutas relativas: img/foto.jpg no C:\imagenes\foto.jpg
Los estilos CSS no se aplicanVerifica que el enlace al CSS sea correcto y que el archivo esté en la carpeta correcta.
Error 404 (página no encontrada)Asegúrate de que el archivo existe y que el nombre es exacto (mayúsculas/minúsculas).
El menú no funcionaVerifica que los enlaces apunten a los archivos correctos. GitHub Pages es sensible a mayúsculas.
